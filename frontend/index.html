CipherGate

A confidential pre-execution policy gate for Safe-compatible treasury intents.

1. Commit the Safe action and submit an encrypted intent

The action fields below are bound into one on-chain commitment before policy evaluation. They will be visible in the exported Safe JSON and in any eventual public Safe transaction. After submission, these fields are locked.

Safe action (public at export)

Before submission and export, CipherGate checks that the Safe address has contract code and that its live nonce() equals the form value. The nonce is then bound only by the CipherGate commitment and JSON metadata: Safe Transaction Builder does not enforce it on import, so owners must check it again. This is an advisory adapter, not a Safe Guard or Module; execution-time enforcement requires a Guard or Module.

Confidential policy inputs

Amount, risk, and flags are sent over TLS to the trusted Nox Gateway. Only encrypted handles and proofs are submitted on-chain. A fresh random opaque audit ID is generated locally; only its hash is public.

No intent submitted.

No Safe action committed.

2. Evaluate and disclose

Evaluation requires the owner/auditor. The normalized result is publicly decryptable; publication verifies Nox's proof on-chain, so any account may relay it.

3. Export Safe Transaction Builder batch

Export is enabled only after CipherGate verifies a PASS proof for the exact committed action, confirms that its deadline has not expired, and rechecks the Safe's live nonce. The JSON contains the public Safe address, destination, value, calldata, nonce metadata, and deadline, but the nonce remains advisory at import. CipherGate never signs or executes it.
